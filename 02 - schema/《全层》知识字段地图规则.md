# 全层知识字段地图规则

版本：v0.2.0

生命周期：Draft

日期：2026-06-21

适用范围：`00 - raw`、`01 - wiki`、`02 - schema`

## 1. 规则边界

本规则只做：

1. 规定 raw、wiki、schema 三层知识文件的 frontmatter 字段地图。
2. 规定每个字段的主执行者和用户复核门槛。
3. 规定字段默认值、占位值和层级适用方式。
4. 规定用户只复核关键判断，不手工维护完整字段。

本规则不做：

1. 不判断 inbox 材料目标目录。
2. 不判断 raw 是否进入 wiki。
3. 不执行 raw 到 wiki 编译。
4. 不执行 GitHub、Supabase 或 GBrain DB 同步。

## 2. 字段维护原则

完整字段由自动化维护。

字段先由 Agent、系统或流程生成。

用户只复核以下关键判断：

| 用户复核项 | 原因 |
| --- | --- |
| `wiki_page_type` | 决定 wiki 页面形态。 |
| `domain_hint` | 决定知识归属。 |
| `trust_level` | 决定是否可作为正式知识。 |
| `evidence_level` | 决定回答时能否作为依据。 |
| `visibility` | 涉及权限范围。 |
| `sensitivity_level` | 涉及安全边界。 |
| `supersedes` | 涉及旧知识替代。 |

每个字段只能有一个主执行者。

用户不是字段主维护者。

Agent 可以生成建议值，系统或流程负责落值，用户负责复核高风险判断。

用户复核结果写入 `review_status`、`verified_by`、`last_verified_at` 或对应流程日志，不改变字段主执行者。

不知道的字段写 `unknown`。

不适用的字段写 `not_applicable`。

列表为空写 `[]`。

## 3. 维护者类型

| 维护者 | 含义 |
| --- | --- |
| 用户复核 | 用户复核会影响归属、可信度、权限或正式结论的判断，不直接承担字段日常维护。 |
| Agent 推断 | Agent 根据正文语义生成字段建议值。 |
| 系统生成 | 时间、ID、版本、hash、默认状态等机械字段。 |
| 流程写入 | inbox 路由、raw 编译、wiki 验收、同步任务把字段值落盘。 |
| 规则管理员 | schema、权限、同步策略等高风险字段。 |

## 4. 字段组

### 4.1 知识身份

| 字段 | 含义 | 主执行者 |
| --- | --- | --- |
| `schema_version` | frontmatter 字段契约版本。 | 系统生成 |
| `knowledge_id` | 稳定知识 ID。 | 系统生成 |
| `title` | 知识标题。 | Agent 拟稿 / 系统生成 |
| `knowledge_layer` | 知识层：`raw`、`wiki`、`schema`。 | 系统根据路径写入 |
| `wiki_page_type` | wiki 页面形态。raw 和 schema 使用 `not_applicable`。 | Agent 推断，用户复核关键项 |
| `process_level` | 流程页等级：指南或 SOP。非流程页使用 `not_applicable`。 | Agent 推断，用户复核关键项 |
| `material_type` | 材料类型。wiki 和 schema 使用 `not_applicable`。 | Agent 推断 |

### 4.2 来源溯源

| 字段 | 含义 | 主执行者 |
| --- | --- | --- |
| `source_type` | 来源类型。 | 系统按入口写入 |
| `source_ref` | URL、对话编号、文件路径、会议记录编号等来源引用。 | 系统生成 |
| `source_title` | 原始来源标题。 | 系统提取 / Agent 提取 |
| `source_author` | 原作者或来源主体。 | 系统提取 / Agent 提取 |
| `captured_at` | 进入知识系统的时间。 | 系统生成 |
| `captured_by` | 捕获者。 | 系统按入口写入 |

### 4.3 路由与分类

| 字段 | 含义 | 主执行者 |
| --- | --- | --- |
| `domain_hint` | 领域归属。 | Agent 推断，流程写入，用户复核关键项 |
| `tags` | 正文主题标签，禁止空格。 | Agent 推断 |
| `route_status` | inbox 路由状态。 | 流程写入 |
| `route_rule_version` | 使用的路由规则版本。 | 流程写入 |
| `routed_at` | 路由完成时间。 | 流程写入 |
| `routed_by` | 路由执行者。 | 流程写入 |

### 4.4 生命周期

| 字段 | 含义 | 主执行者 |
| --- | --- | --- |
| `status` | 生命周期状态。 | 流程写入 |
| `review_status` | 审核状态。 | 流程写入 |
| `created_at` | 文件创建时间。 | 系统生成 |
| `updated_at` | 最近更新时间。 | 系统生成 |
| `promoted_at` | 晋升为 active 的时间。 | 流程写入 |
| `promotion_basis` | 晋升依据。 | 流程写入 |
| `archived_at` | 归档时间。 | 流程写入 |
| `deprecated_at` | 废弃时间。 | 流程写入 |
| `deprecated_reason` | 废弃原因。 | 流程写入 |
| `replaced_by` | 替代它的新知识。 | 流程写入 |
| `rejected_reason` | 拒绝原因。 | 流程写入 |

### 4.5 编译关系

| 字段 | 含义 | 主执行者 |
| --- | --- | --- |
| `compile_status` | raw 是否已经参与 wiki 编译。 | 流程写入 |
| `source_refs` | 支撑当前页面结论的 raw 来源引用。 | 编译流程写入 |
| `compiled_from` | 当前 wiki/schema 来源于哪些 raw。 | 编译流程写入 |
| `compiled_to` | 当前 raw 被编译到哪些 wiki。 | 编译流程写入 |
| `compile_rule_version` | 使用的编译规则版本。 | 编译流程写入 |
| `compile_batch_id` | 编译批次编号。 | 编译流程写入 |
| `compiled_at` | 编译完成时间。 | 编译流程写入 |
| `compiled_by` | 编译执行者。 | 编译流程写入 |

### 4.6 可信度

| 字段 | 含义 | 主执行者 |
| --- | --- | --- |
| `trust_level` | 可信等级。 | Agent 推断，流程写入，用户复核关键项 |
| `confidence` | 判断置信度。 | Agent 推断 |
| `evidence_level` | 证据等级。 | Agent 推断，用户复核关键项 |
| `risk_level` | 知识被错误使用时的风险等级。 | Agent 推断，用户复核关键项 |
| `last_verified_at` | 最近验证时间。 | 流程写入 |
| `verified_by` | 验证者。 | 流程写入 |

### 4.7 检索与回答策略

| 字段 | 含义 | 主执行者 |
| --- | --- | --- |
| `retrieval_scope` | 检索范围。 | 规则默认，流程写入，用户复核例外 |
| `retrieval_priority` | 检索优先级。 | 流程写入 |
| `answer_policy` | 回答策略。 | 流程写入 |
| `search_boost` | 检索加权。 | 系统生成 / 流程写入 |

### 4.8 权限治理

| 字段 | 含义 | 主执行者 |
| --- | --- | --- |
| `visibility` | 可见范围。 | Agent 推断，规则管理员配置默认值，用户复核关键项 |
| `owner` | 维护责任人或角色。 | Agent 推断，流程写入 |
| `editable_by` | 可编辑角色列表。 | 规则管理员 |
| `sensitivity_level` | 敏感等级。 | Agent 推断，用户复核关键项 |

### 4.9 版本替代

| 字段 | 含义 | 主执行者 |
| --- | --- | --- |
| `supersedes` | 当前知识替代哪些旧知识。 | Agent 推断，编译流程写入，用户复核关键项 |
| `superseded_by` | 当前知识被哪条新知识替代。 | 编译流程写入 |
| `version` | 当前知识版本。 | 系统生成 / 流程写入 |
| `changelog_ref` | 对应变更记录。 | 流程写入 |

### 4.10 同步审计

| 字段 | 含义 | 主执行者 |
| --- | --- | --- |
| `content_hash` | 正文内容 hash。 | 系统生成 |
| `last_synced_at` | 最近同步到 GBrain DB 时间。 | 同步流程写入 |
| `last_synced_by` | 同步执行者。 | 同步流程写入 |
| `sync_status` | 同步状态。 | 同步流程写入 |
| `sync_error` | 同步失败原因。 | 同步流程写入 |

## 7. 字段同步要求

修改字段、允许值、默认值、主执行者、用户复核门槛或层级默认值时，必须同步检查：

1. 《00 - inbox》路由分发规则.md
2. 《00 - raw》编译状态规则.md
3. 《01 - wiki》知识编译规则.md
4. 《01 - wiki》质量验收规则.md
5. 《全层》Agent 回写工具边界规则.md
6. 《全层》MCP 查询策略规则.md
7. 模板/00 - raw 材料说明卡模板.md
8. 模板/01 - wiki 页面模板.md
9. 模板/02 - schema 规则文件模板.md
10. 自动化脚本/00 - inbox 校验与路由输入 dry-run.ps1
11. 规则变更日志.md
