# PRD 需求拆解 Agent

## 当前结论

1. PRD 需求拆解 Agent 的目标是把模糊需求拆成目标、范围、非目标、用户场景、流程、验收标准和任务切分。
2. 该 Agent 不能直接默认用户的初始表述完整正确，必须先澄清缺失信息和逻辑冲突。
3. 好的输出应能进入产品评审、研发拆解、测试验收和后续项目记忆。
4. 涉及商业模式、权限、安全、财务、合规的需求，必须标记风险并请求用户确认。
5. 该案例适合沉淀为 Complex Task Clarifier Skill、PRD 模板和验收标准模板。

## 适用范围

### 适用

- 将一句话想法转成 PRD 草案。
- 将用户需求拆成 user story、acceptance criteria 和任务清单。
- 发现需求中的矛盾、缺口、范围蔓延和不可验收描述。
- 为 Codex 或研发 Agent 提供明确输入。

### 不适用

- 不替代产品负责人做最终取舍。
- 不自动确认高风险业务规则。
- 不把模糊需求直接转成开发任务。

## 案例字段

| 字段 | 内容 |
| --- | --- |
| Goal | 将模糊需求拆成可评审、可执行、可验收的产品需求。 |
| Input | 用户想法、业务背景、目标用户、当前流程、约束。 |
| Context | 产品策略、项目目标、历史决策、设计规范、技术边界。 |
| Tools | 文档生成、知识库检索、任务拆分、模板填充。 |
| Output | PRD、用户故事、验收标准、风险清单、待确认问题。 |
| Quality | 边界清晰，可验收，有证据，缺口明确。 |
| Risk | 默认认同错误需求、范围蔓延、遗漏权限安全、任务不可验收。 |
| Asset | PRD 模板、需求澄清 Skill、验收标准模板、项目记忆。 |
