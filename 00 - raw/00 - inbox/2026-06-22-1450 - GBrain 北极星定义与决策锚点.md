## 原始记录内容

这个定义把 GBrain 从“知识库 / 工作环境”提升成三位一体：

GBrain 是一个让人和 AI 持续变聪明的工作环境，也是 Agent 的记忆大脑，同时是个人和企业的知识复利资产库。

它的核心特征是：

1. 可沉淀。
2. 可追溯。
3. 可演化。

它解决的不是“怎么存资料”，而是以下四个更深的问题：

1. Agent 的连续性问题：跨设备、跨线程、跨时间后，Agent 不能忘记任务背景、当前进度、下一步和历史决策。
2. 知识资产复利问题：有价值的经验、决策、踩坑、处置日志，不应随一次对话结束而消失；未来遇到相似信号时，Agent 应该能主动调取历史案例、策略和结果。
3. Agent 从实习生成长为老员工的问题：它要懂流程、规则、背景、边界、验收标准，而不是每次都从零开始问、猜、试错。
4. 信息生命周期治理问题：信息不是静态文本。它会增长、连接、被验证、失效、腐化。所以它需要状态：可决策、待验证、冲突、过期、尘封、替代，而不是简单地“存进去”。

这也解释了为什么 GBrain 必须有 raw / wiki / schema 这样的分层。

GBrain 不是在建一个“资料仓库”，而是在建一个长期运转的认知治理系统：

```text
先记录
再核查
再编译
再使用
再更新
```

这应作为后续理解 GBrain 的北极星。

## 工程含义

后续设计、实现、同步、回写、查询、schema 治理和 Agent 行为设计都必须用这条定义做校准。

如果某个方案只优化“资料存储”，但无法支撑跨线程续作、历史经验复利、老员工式执行或信息生命周期治理，则该方案偏离 GBrain 的目标。

如果某个动作能让信息可沉淀、可追溯、可演化，并能帮助人和 AI 在未来更聪明地工作，则它更接近 GBrain 的目标。
