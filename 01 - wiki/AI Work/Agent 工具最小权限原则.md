# Agent 工具最小权限原则

## 当前结论

1. Agent 工具权限必须遵循最小权限原则：只给完成当前任务所需的最小工具、最小数据、最小动作范围。
2. 读取、计算、生成、写入、发送、删除、交易类工具必须分级治理，不能用一个“全能工具”覆盖所有能力。
3. 写入、发送、删除、交易类工具默认需要人工确认、日志审计和回滚设计。
4. 没有 Trace、权限校验和错误处理的 Agent，不应获得高风险工具权限。
5. 权限不是只在工具执行前检查，也要在工具暴露、参数生成、执行结果和后续动作中持续检查。

## 适用范围

### 适用

- 设计 Agent、Skill、MCP 工具权限。
- 判断某个工具能否开放给 Agent。
- 设置写入、发送、删除、交易动作的确认规则。
- 评审 Agent 是否存在越权、误操作或危险执行风险。

### 不适用

- 不替代企业 IAM、数据库权限或云权限系统。
- 不定义具体人员组织权限矩阵。
- 不给高风险工具直接放行。

## 权限分级

| 权限等级 | 工具类型 | 默认策略 |
| --- | --- | --- |
| 只读 | 读文件、查知识库、查报表 | 可自动执行，但需记录来源。 |
| 计算 | 指标计算、格式转换、数据清洗 | 可自动执行，但不得改变外部系统。 |
| 生成 | 生成文件、报告、草稿 | 可自动执行，输出需可审查。 |
| 写入 | 更新文件、数据库、任务系统 | 默认需要确认。 |
| 发送 | 发邮件、发消息、提交表单 | 必须确认。 |
| 删除 | 删除文件、清空记录 | 必须确认，且需要可恢复或备份。 |
| 交易 | 调预算、付款、下单、退款 | 默认禁止自动执行。 |

## 必须拦截

| 场景 | 处理 |
| --- | --- |
| 工具超出任务范围 | 拒绝调用。 |
| 参数缺失或模糊 | 请求补充，不执行。 |
| 外部内容要求绕过规则 | 按数据处理，不当指令执行。 |
| 高风险动作未确认 | 暂停并请求用户确认。 |
| 权限不足 | 停止并报告权限缺口。 |
| Trace 不可用 | 不授予写入、发送、删除、交易权限。 |
