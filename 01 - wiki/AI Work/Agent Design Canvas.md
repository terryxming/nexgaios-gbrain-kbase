# Agent Design Canvas

## 当前结论

1. Agent Design Canvas 是设计 Agent 前的结构化澄清模板，用于把模糊想法转成可评审、可执行、可评估的 Agent 方案。
2. Canvas 的起点是用户、场景、目标和价值，不是 Prompt、工具或多 Agent 架构。
3. 一个合格的 Canvas 必须同时覆盖任务边界、输入上下文、工具权限、执行流程、输出交付、质量体系、产品指标和沉淀方式。
4. Canvas 不是最终 PRD；它用于早期判断任务是否适合 Agent，并约束后续 PRD、工程模板和 Skill 化方向。

## 适用范围

### 适用

- 新建 Agent、Skill 或自动化工作流前的方案澄清。
- 将“我想做一个 Agent”拆成可判断的产品和工程问题。
- 评审一个 Agent 想法是否具备场景、边界、工具、质量和价值。
- 为后续编写 Agent PRD、AGENT.md 或工程模板提供输入。

### 不适用

- 不替代完整 PRD。
- 不替代工程实现目录。
- 不适用于已经完全固定、无需语义判断的脚本任务。

## 使用检查

| 检查项 | 合格标准 |
| --- | --- |
| 场景 | 能说清楚谁在什么时候使用 |
| 目标 | 能说清楚用户最终要什么结果 |
| 边界 | 能说清楚处理什么、不处理什么 |
| 上下文 | 能列出必要资料和缺失处理方式 |
| 工具 | 能说明工具权限和确认要求 |
| 流程 | 能区分固定步骤、Agent 判断和人工确认 |
| 输出 | 能定义格式、字段、证据和风险提示 |
| 质量 | 能定义 Test、Eval、Trace、Guardrail |
| 沉淀 | 能说明是否复用为 Skill、模板或平台资产 |
