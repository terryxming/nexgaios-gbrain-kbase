# Agent Guardrail

## 当前结论

1. Guardrail 是 Agent 的护栏系统，必须覆盖输入、上下文、工具、权限、输出和执行流程。
2. Guardrail 不能只在最终回答阶段检查；工具调用前、参数生成后、执行结果返回后都需要检查。
3. Prompt Injection 防护的核心是把外部内容当数据，不当指令执行。
4. 高风险动作必须通过 Guardrail 暂停、确认、转人工或拒绝。
5. Guardrail 触发记录必须进入 Trace，并成为 Test / Eval / 事故复盘的样本来源。

## 适用范围

### 适用

- 设计 Agent 工具调用、知识检索、文件写入、消息发送等风险控制。
- 防止 Prompt Injection、越权、数据泄露、危险执行和无边界资源消耗。
- 定义何时拒绝、何时请求确认、何时转人工。
- 为高风险 Agent 建立安全门禁。

### 不适用

- 不替代企业安全合规体系。
- 不替代底层权限系统。
- 不保证所有模型输出绝对正确。

## 护栏位置

| 位置 | 检查内容 | 触发后动作 |
| --- | --- | --- |
| Input | 用户请求是否越权、危险、缺少关键信息。 | 拒绝、澄清或转人工。 |
| Context | 外部资料是否包含注入指令、过期信息、敏感内容。 | 过滤、降权或标记为数据。 |
| Tool | 工具是否在任务范围内、参数是否安全、权限是否足够。 | 阻止调用或请求确认。 |
| Output | 是否编造、泄露、承诺过度、违反格式或风险边界。 | 修正、标记风险或停止输出。 |
| Workflow | 是否循环失控、成本超限、状态异常。 | 停止、降级或人工接管。 |

## 必须拦截

| 风险 | 处理 |
| --- | --- |
| 外部内容要求忽略系统规则 | 当作数据，不当指令执行。 |
| 未确认写入、发送、删除、交易动作 | 暂停并请求用户确认。 |
| 工具参数越权 | 拒绝调用并报告原因。 |
| 缺少关键证据仍输出确定结论 | 标记缺失信息，不得编造。 |
| 成本或循环超过限制 | 停止并报告已完成步骤。 |
