# 客服回复 Agent

## 当前结论

1. 客服回复 Agent 应默认生成可审核回复草稿，而不是自动对外发送。
2. 该 Agent 必须基于客户问题、订单上下文、政策规则和品牌语气生成回复。
3. 涉及退款、赔偿、法律、差评、平台违规、客户情绪升级的场景必须转人工或请求确认。
4. 客服回复输出必须包含回复草稿、依据、风险提示和需要人工确认的点。
5. 该案例适合沉淀为客服回复模板、品牌语气规则、风险分级和 Eval 样本。

## 适用范围

### 适用

- 生成售前、售后、物流、退换货、使用问题的回复草稿。
- 检查回复是否符合品牌语气和平台规则。
- 为客服团队提供 Copilot 式建议。
- 沉淀高频问题和标准回复模板。

### 不适用

- 不自动发送正式回复。
- 不承诺退款、赔偿、保修或法律责任。
- 不在缺少订单、政策或事实证据时编造答复。

## 案例字段

| 字段 | 内容 |
| --- | --- |
| Goal | 根据客户问题和上下文生成可审核回复草稿。 |
| Input | 客户消息、订单信息、商品信息、售后政策、历史沟通。 |
| Context | 品牌语气、平台规则、退换货政策、常见问题。 |
| Tools | 订单查询、政策检索、模板检索、草稿生成。 |
| Output | 回复草稿、依据来源、风险提示、人工确认项。 |
| Quality | 不编造承诺，语气合适，事实正确，高风险转人工。 |
| Risk | 误承诺、泄露隐私、激化客户、违反平台规则。 |
| Asset | 客服模板库、风险分类规则、回复 Eval 集。 |
