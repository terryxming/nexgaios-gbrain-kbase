# 全层知识生产流水线操作手册

版本：v0.4.6

生命周期：Draft

日期：2026-06-22

适用范围：网页剪藏、用户手写、Agent 回写、inbox 路由、raw 编译、wiki 晋升、GitHub 知识库同步、GBrain DB 同步、MCP 查询

## 1. 手册定位

本手册是 Nexgaios GBrain 知识生产流水线的主入口。

本手册只负责调度，不承载全部细则。

Agent 执行知识库任务时，先读本手册，再按当前阶段读取对应细则。

## 2. 渐进式披露原则

规则按流水线阶段披露。

Agent 只读取当前任务需要的规则细则。

禁止为了执行单个阶段，把所有 schema 规则一次性塞进上下文。

当任务跨阶段时，按执行顺序逐段读取细则。

## 3. 知识生产主线

第一版知识飞轮固定为：

```text
知识入口
-> 00 - raw/00 - inbox
-> inbox 接收与字段修复
-> inbox 路由分发
-> 00 - raw 领域目录
-> raw 编译状态管理
-> raw 到 wiki 编译
-> 01 - wiki candidate
-> wiki 质量验收
-> 01 - wiki active
-> GitHub 知识库同步
-> GBrain DB 同步
-> MCP 查询和 Agent 使用
-> Agent 回写再次进入知识入口
```

规则服务于这条主线。不能脱离流程单独解释字段、目录、状态或同步行为。

## 6. 全局约束

所有入口都必须进入 inbox 接收流程。

禁止绕过 inbox 直接进入领域 raw 目录。

禁止绕过 raw 直接生成 active wiki。

Agent 回写必须先进入 GBrain DB raw 即时记忆，再物化到本地 inbox。

所有 raw、wiki、schema Markdown frontmatter 全量展示 14 个字段。字段定义、枚举值和生成规则以 `流水线细则/01 - 知识入口与字段规则.md` 为准。

schema 是规则来源，不是业务事实来源。

raw 可以作为即时记忆和证据来源，未编译 raw 不得伪装成正式结论。

wiki active 是默认回答来源。

## 7. 冲突处理

流程调度冲突时，以本手册为准。

阶段细则冲突时，停止执行并要求用户确认。

脚本、模板与规则冲突时，以规则为准，并同步修改脚本或模板。

旧主题规则文件不保留兼容入口。历史规则变化只通过 `规则变更日志.md` 追溯。

## 8. 执行顺序

Agent 执行知识任务时，按以下顺序读取：

1. 本手册。
2. 当前阶段对应的流水线细则。
3. 当前阶段引用的脚本、模板或 skill。
4. 规则变更日志。

跨阶段任务必须在进入下一阶段前读取下一阶段细则。
